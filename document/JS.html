Abstract

This document provides an explicit JavaScript API for interacting with WebAssembly.

This API is, initially, the only API for accessing WebAssembly [WEBASSEMBLY] from the Web Platform, through a bridge to explicitly construct modules from ECMAScript [ECMASCRIPT].

In future versions, WebAssembly be loaded and run directly from an HTML <script type='module'> tag—and any other Web API that loads ES6 modules via URL—as part of ES6 Module integration.)

Note: WebAssembly JS API declaration file for TypeScript can be found here which enable autocompletion and make TypeScript compiler happy.

1. The WebAssembly Object

The following IDL uses a couple features which are not in IDL currently:

  • To support constructors which are properties of the WebAssembly object, namespaces define interfacemembers. In the ECMAScript binding, these are writable, configurable, non-enumerable properties of the namespace. Inner interfaces inherit the exposure set of the outer namespace.

  • The @@toStringTag of interfaces which are declared in a namespace is the name of the outer namespace, followed by ., followed by the name of the interface.

  • The [Nowrap] extended attribute to an integer type throws a RangeError if the Number is greater than or less than the range which can be accurately represented by the type.

    [Exposed=(Window,Worker)]
namespace WebAssembly {
  boolean validate(BufferSource bytes);
  Promise compile(BufferSource bytes);
  
  Promise instantiate(
      BufferSource bytes, optional ImportObject importObject);

  Promise instantiate(
      Module moduleObject, optional ImportObject importObject);
};
To parse a WebAssembly binary module from a BufferSource buffer, return a module AST whose binary format is bytewise identical to buffer. If no such AST can be found, throw a WebAssembly.CompileError exception.
To validate a WebAssembly module from a WebAssembly AST ast, perform the following steps:

  1. Attempt to prove ⊢ ast from the logic in WebAssembly module validation. If it cannot be derived, throw a WebAssembly.CompileError exception.

  2. Validate any import names (for modules and individual imports) as UTF-8. If the module has a custom name section, validate any module names, function names and local names in the section as UTF-8. If validation fails, throw a WebAssembly.CompileError exception.

  3. Return.

The validate(bytes) method, when invoked, performs the following steps:

  1. Let ast be the result of parse the WebAssembly binary. If this throws an exception, catch it, and return false.

  2. validate the WebAssembly module ast. If this throws an exception, catch it, and return false.

  3. Return true.

To compile a WebAssembly module from a BufferSource bytes, perform the following steps:

  1. Let ast be the result of parse the WebAssembly binary bytes.

  2. validate the WebAssembly module ast.

  3. Return a new WebAssembly.Module object is returned with [Module] set to the validated module ast.

The compile(bytes) method, when invoked, performs the following steps:

  1. Let stableBytes be a copy of the bytes held by the buffer source bytes.

  2. Return a new promise promise and perform the following steps in parallel:

    1. Compile a WebAssembly module from stableBytes, and let module be the result. If this throws an exception, catch it, and reject promise with the exception.

    2. Resolve promise with module.

To instantiate a WebAssembly module from a WebAssembly.Module moduleObject and imports importObject, perform the following steps:

  1. Let module be moduleObject.[[Module]].

  2. If module.imports is not an empty list, and importObject is undefined, throw a TypeError exception.

  3. Let funcs be an empty List of callable JavaScript objects.

  4. Let memories be an empty List of WebAssembly.Memory objects.

  5. Let tables be an empty List of WebAssembly.Table objects.

  6. Let imports be an empty List.

  7. For each import import in module.imports,

    1. Let o be the resultant value of performing [Get](http://tc39.github.io/ecma262/#sec-get-o-p)(importObject, [i.module_name](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/ast.ml#L170)).

    2. If Type(o) is not Object, throw a [TypeError](https://tc39.github.io/ecma262/#sec-native-error-types-used-in-this-standard-typeerror).

    3. Let v be the value of performing [Get](http://tc39.github.io/ecma262/#sec-get-o-p)(o, [i.item_name](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/ast.ml#L171))

    4. If i is a function import:

    5. If [IsCallable(v)](https://tc39.github.io/ecma262/#sec-iscallable) is false, throw a WebAssembly.LinkError.

    6. If v is an Exported Function Exotic Object:

      1. (The signature of v.[[Closure]] is checked against the import’s declared [func_type](https://github.com/WebAssembly/design/blob/master/BinaryEncoding.md#func_type) by Eval.init below.)

      2. Let closure be v.[[Closure]].

    7. Otherwise:

      1. Let closure be a new host function value of the given signature and the following behavior:

      2. If the signature contains an i64 (as argument or result), the host function immediately throws a [TypeError](https://tc39.github.io/ecma262/#sec-native-error-types-used-in-this-standard-typeerror) when called.

      3. Otherwise, the host function calls v with an undefined receiver and WebAssembly arguments coerced to JavaScript arguments via [ToJSValue](#tojsvalue). The result is returned by coercing via [ToWebAssemblyValue](#towebassemblyvalue).

    8. Append v to funcs.

    9. Append closure to imports.

    10. If i is a global import:

    11. Assert: the global is immutable by MVP validation constraint.

    12. If the global_type of i is i64 or Type(v) is not Number, throw a WebAssembly.LinkError.

    13. Append [ToWebAssemblyValue](#towebassemblyvalue)(v) to imports.

    14. If i is a memory import:

    15. If v is not a [WebAssembly.Memory object](#webassemblymemory-objects), throw a WebAssembly.LinkError.

    16. (The imported Memory's length and maximum properties are checked against the import’s declared [memory_type](https://github.com/WebAssembly/design/blob/master/BinaryEncoding.md#memory_type) by Eval.init below.)

    17. Append v to memories.

    18. Append v.[[Memory]] to imports.

    19. Otherwise (i is a table import):

    20. If v is not a [WebAssembly.Table object](#webassemblytable-objects), throw a WebAssembly.LinkError.

    21. (The imported Table's length, maximum and element properties are checked against the import’s declared [table_type](https://github.com/WebAssembly/design/blob/master/BinaryEncoding.md#table_type) by Eval.init below.)

    22. Append v to tables.

    23. Append v.[[Table]] to imports.

    24. For each index i of v.[[Table]]:

      1. Let e be the ithe element of v.[[Table]].

    25. If e is a [closure](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/instance.ml#L7) c:

      1. Append the ith element of v.[[Values]] to funcs.

  8. Let instance be the result of creating a new [instance](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/instance.ml#L17) by calling [Eval.init](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/eval.ml#L416) given module and imports. If this terminates with a Link error, throw a WebAssembly.LinkError; if it causes a trap, throw a WebAssembly.RuntimeError; all other exceptions are propagated to the caller. Among other things, this function performs the following observable steps:

    • If, after evaluating the offset initializer expression of every Data and Element Segment, any of the segments do not fit in their respective Memory or Table, throw a WebAssembly.LinkError.

    • Apply all Data and Element segments to their respective Memory or Table in the order in which they appear in the module. Segments may overlap and, if they do, the final value is the last value written in order. Note: there should be no errors possible that would cause this operation to fail partway through. After this operation completes, elements of instance are visible and callable through imported tables, even if start fails.

    • If a [start](Modules.md#module-start-function) is present, it is evaluated. Any errors thrown by start are propagated to the caller.

  9. The following steps are performed _before_ the start function executes:

    1. For each table 't' in [instance.tables](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/instance.ml#L17):

    2. If there is no element in tables whose table.[[Table]] is t:

      1. Let table be a new WebAssembly.Table object with [Table] set to t and [Values] set to a new list of the same length all whose entries are null.

      2. Append table to tables.

    3. Otherwise:

      1. Let table be the element in tables whose table.[[Table]] is t

    4. (Note: At most one WebAssembly.Table object is created for any table, so the above table is unique, even if there are multiple occurrances in the list. Moreover, if the item was an import, the original object will be found.)

    5. For each index i of t:

      1. Let c be the ith element of t

      2. If c is a [closure](https://github.com/WebAssembly/spec/blob/master/interpreter/spec/instance.ml#L7) c:

        1. If there is an Exported Function Exotic Object in funcs whose [[Closure]] equals c:

          1. Let func be that function object.

        2. (Note: At most one wrapper is created for any closure, so func is uniquely determined. Moreover, if the item was an import that is already an Exported Function Exotic Object, then the original function object will be found. For imports that are regular JS functions, a new wrapper will be created.)

        3. Otherwise:

          1. Let func be an Exported Function Exotic Object created from c.

          2. Append func to funcs.

        4. Set the ith element of table.[[Values]] to func.

    (Note: The table and element function objects created by the above steps are only observable for tables that are either imported or exported.)

  10. Let exports be a list of (string, JS value) pairs that is mapped from each external value e in instance.exports as follows:

    1. If e is a closure c:

    2. If there is an Exported Function Exotic Object func in funcs whose func.[[Closure]] equals c, then return func.

    3. (Note: At most one wrapper is created for any closure, so func is unique, even if there are multiple occurrances in the list. Moreover, if the item was an import that is already an Exported Function Exotic Object, then the original function object will be found. For imports that are regular JS functions, a new wrapper will be created.)

    4. Otherwise:

      1. Let func be an Exported Function Exotic Object created from c.

      2. Append func to funcs.

      3. Return func.

    5. If e is a global v:

    6. Assert: the global is immutable by MVP validation constraint.

    7. If v is an i64, throw a WebAssembly.LinkError.

    8. Return [ToJSValue](#tojsvalue)(v).

    9. If e is a memory m:

    10. If there is an element memory in memories whose memory.[[Memory]] is m, then return memory.

    11. (Note: At most one WebAssembly.Memory object is created for any memory, so the above memory is unique, even if there are multiple occurrances in the list. Moreover, if the item was an import, the original object will be found.)

    12. Otherwise:

      1. Let memory be a new WebAssembly.Memory object created via [CreateMemoryObject](#creatememoryobject) from m.

      2. Append memory to memories.

      3. Return memory.

    13. Otherwise e must be a table t:

    14. Assert: There is an element table in tables whose table.[[Table]] is t.

    15. Return that table.

  11. Let exportsObject be a new frozen plain JS object with [Prototype] set to Null and with properties defined by mapping each export in exports to an enumerable, non-writable, non-configurable data property. Note: the validity and uniqueness checks performed during module validation ensure that each property name is valid and no properties are defined twice.

  12. Let instanceObject be a new WebAssembly.Instance object setting the internal [[Instance]] slot to instance and the [[Exports]] slot to exportsObject.

  13. Return instanceObject.

The instantiate(bytes, importObject) method, when invoked, returns a new promise promise and performs the following steps in parallel:

  1. Let stableBytes be a copy of the bytes held by the buffer source bytes.

  2. Return a new promise promise and perform the following steps in parallel:

    1. Construct a WebAssembly module from stableBytes, and let module be the result. If this throws an exception, catch it, and reject promise with the exception.

    2. Instantiate the WebAssembly module module importing importObject, and let instance be the result. If this throws an exception, catch it, and reject promise with the exception.

    3. Let result be a WebAssemblyInstantiatedSource dictionary with module set to module and instance set to instance.

    4. Resolve promise with result.

The instantiate(moduleObject, importObject) method, when invoked, returns a new promise promise and performs the following steps in parallel:

  1. Instantiate the WebAssembly module module importing importObject, and let instance be the result. If this throws an exception, catch it, and reject promise with the exception.

  2. Resolve promise with instance.

Note: A follow-on streaming API is documented in the WebAssembly design repository.

The name of the export export in the module module is defined as follows:

  1. If export is of the form 𝖿𝗎𝗇𝖼 funcidx and module has a name section with a function names subsection,

    1. Within funcnamesubsec, search for an entry nameassoc of the form funcidx name. If such an entry is found, return name decoded as a UTF-8 into a JavaScript String.

  2. Return the empty string.

The exports(module) method, when invoked, performs the following steps:

  1. Let ast be module.[Module].

  2. Let exports be an empty List.

  3. For each export in ast.exports:

    1. Let name be export.name as a String.

    2. Let kind be

      • "function" if export.desc is of the form 𝖿𝗎𝗇𝖼 funcidx

      • "table" if export.desc is of the form 𝗍𝖺𝖻𝗅𝖾 tableidx

      • "memory" if export.desc is of the form 𝗆𝖾𝗆 memidx

      • "global" if export.desc is of the form 𝗀𝗅𝗈𝖻𝖺𝗅 globalidx

    3. Let obj be ObjectCreate(%ObjectPrototype%).

    4. Perform CreateDataProperty(obj, "name", name).

    5. Perform CreateDataProperty(obj, "kind", kind).

    6. Append obj to the end of exports.

  4. Return CreateArrayFromList(exports).

The imports(module) method, when invoked, performs the following steps:

  1. Let ast be module.[Module].

  2. Let imports be an empty List.

  3. For each imports in ast.imports:

    1. Let module be import.module as a String.

    2. Let name be import.name as a String.

    3. Let kind be

      • "function" if export is of the form 𝖿𝗎𝗇𝖼 typeidx

      • "table" if export is of the form 𝗍𝖺𝖻𝗅𝖾 tabletype

      • "memory" if export is of the form 𝗆𝖾𝗆 memtype

      • "global" if export is of the form 𝗀𝗅𝗈𝖻𝖺𝗅 globaltype

    4. Let obj be ObjectCreate(%ObjectPrototype%).

    5. Perform CreateDataProperty(obj, "module", module).

    6. Perform CreateDataProperty(obj, "name", name).

    7. Perform CreateDataProperty(obj, "kind", kind).

    8. Append obj to the end of imports.

  4. Return CreateArrayFromList(imports).

1.1. Error objects

WebAssembly defines three Error classes. Analogous to DOMException, WebAssembly errors have the following custom bindings:

  • Unlike normal interface types, the interface prototype object for DOMException must have as its [Prototype] the intrinsic object %ErrorPrototype%.

  • If an implementation gives native Error objects special powers or nonstandard properties (such as a stack property), it should also expose those on DOMException instances.

  • The constructor and properties of WebAssembly errors is as specified for NativeErrors.

  interface CompileError { }
  interface LinkError { }
  interface RuntimeError { }
}

2. Error condition mappings to JavaScript

Running WebAssembly programs encounter certain events which halt execution of the WebAssembly code. WebAssembly code (currently) has no way to catch these conditions and thus an exception will necessarily propagate to the enclosing non-WebAssembly caller (either the browser or JavaScript) where it is handled like a normal JavaScript exception.

If WebAssembly calls JavaScript via import and the JavaScript throws an exception, the exception is propagated through the WebAssembly activation to the enclosing caller.

Because JavaScript exceptions can be handled, and JavaScript can continue to call WebAssembly exports after a trap has been handled, traps do not, in general, prevent future execution.

2.1. Traps

Whenever WebAssembly semantics specify a trap, a WebAssembly.RuntimeError object is thrown to the enclosing JavaScript.

2.2. Stack Overflow

Whenever a stack overflow occurs in WebAssembly code, the same class of exception is thrown as for a stack overflow in JavaScript. The particular exception here is implementation-defined in both cases.

2.3. Out of Memory

Whenever validation, compilation or instantiation run out of memory, the same class of exception is thrown as for out of memory conditions in JavaScript. The particular exception here is implementation-defined in both cases.
